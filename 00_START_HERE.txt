A production-ready, automated E2E test plan generator for Comcast Atlas retention 
offers. Transforms raw CastleKeep offer data → Professional Excel workbook with 
comprehensive positive/negative test cases in under 2 seconds.

KEY FEATURES:
  ✅ Automatic offer classification (Standard vs. Strategic)
  ✅ Intelligent test case generation (5-7 tests per offer)
  ✅ Professional 5-sheet Excel workbook
  ✅ Geographic scope detection (National vs. Regional)
  ✅ XM discount validation for Strategic offers
  ✅ CastleKeep Bridge integration
  ✅ Writer Agent playbook examples
  ✅ Comprehensive documentation (4 guides + code comments)
  ✅ Tested and verified with sample data

  TEST CASE COUNTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GROUP A (Standard Offers)
  Without CUSTOM_FIELD03_HSE tag
  → 2 Positive Tests
     • RETENTION_TELESALES channel
     • RETENTION-TEST_TELES channel
  → 3 Negative Tests
     • Wrong sales channel (ONLINE)
     • Business/Commercial customer
     • Wrong division (skipped if National)
  = 5 tests per offer

GROUP B (Strategic Offers)
  With CUSTOM_FIELD03_HSE:1xxx tag (position 1 = '1')
  → 3 Positive Tests
     • Correct tag via RETENTION_TELESALES
     • Correct tag via RETENTION-TEST_TELES
     • XM-New with discount
  → 4 Negative Tests
     • Missing CUSTOM_FIELD03_HSE tag
     • Tag with position 1 = '0' (wrong value)
     • Correct tag but wrong channel
     • Correct tag but wrong division (skipped if National)
  = 7 tests per offer

EXAMPLE
  2 offers (1 Group A + 1 Group B) = 5 + 7 = 12 test cases

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SHEET 1: EXECUTIVE SUMMARY
  High-level overview with:
  • Report date & ticket number
  • Request name & business strategy
  • Geographic scope
  • Offer counts (total, Group A, Group B)
  • Test case metrics (positive, negative, total)

SHEET 2: OFFER SUMMARY
  One row per unique offer showing:
  • Offer ID, Name, Start/End dates
  • Geographic scope
  • Customer eligibility
  • Test case counts (pos, neg, total)
  + TOTALS row

SHEET 3: POSITIVE TEST CASES
  All valid scenario tests (15 columns):
  Test ID | Test Type | Offer ID | Offer Name | Description | Customer Status | 
  Customer Type | Customer Tags | Sales Channel | Division | Tenure Days | 
  Existing Services | Speed Tier | Expected Result | Validation Points
  
  Color: Green background (#E2EFDA) for easy scanning
  Frozen: Header row (row 1)

SHEET 4: NEGATIVE TEST CASES
  All failure scenario tests (16 columns):
  [Same 15 columns as POSITIVE] + Failure Reason

  Color: Red/Orange background (#FCE4D6) for easy scanning
  Frozen: Header row (row 1)

SHEET 5: ALL TEST CASES
  Interleaved positive and negative tests, sorted by offer:
  [Positive tests for OFR-1]
  [Negative tests for OFR-1]
  [Positive tests for OFR-2]
  [Negative tests for OFR-2]
  ... etc

  Color: Light green (positive) or light red (negative)
  Frozen: Header row (row 1)
